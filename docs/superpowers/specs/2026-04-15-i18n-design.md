# Chrome AI Assistant - 国际化功能设计

**创建时间**: 2026-04-15
**状态**: 待实现

---

## 概述

为 Chrome AI Assistant 扩展实现完整的国际化功能，支持中文（zh-CN）和英文（en-US）切换，包括自定义 UI 文本和 Element Plus UI 组件库的内置文本。

---

## 当前问题

- Settings 面板有语言选项（zh-CN / en-US），但设置后无实际效果
- 所有 UI 文本硬编码为英文
- Element Plus 组件（日期选择器、分页等）始终显示英文
- 缺少国际化框架和语言文件

---

## 设计方案

### 技术选型

采用 `vue-i18n` + `Element Plus locale` 标准方案：

- **vue-i18n**: Vue 3 官方推荐的国际化管理库
- **Element Plus locale**: 内置支持多语言切换
- **自建语言文件**: 合理组织翻译文本结构

### 架构

**核心组件**：

- `vue-i18n` - Vue 3 国际化库
- `Element Plus locale` - UI 组件库内置文本切换
- `locales/` - 语言文件目录（zh-CN.ts、en-US.ts）
- `useLocale` composable - 语言切换逻辑封装

**集成方式**：

- `main.ts` 初始化 i18n 并配置 Element Plus locale
- App.vue 监听 storage 语言设置变化
- 所有组件使用 `$t()` 或 `t()` 函数获取翻译文本

## 数据流

### 语言切换流程

1. 用户在 Settings 选择语言 → 保存到 `storage.config.language`
2. App.vue 监听 config 变化 → 更新 i18n locale
3. Element Plus 接收 locale prop → UI 组件文本切换
4. 所有 Vue 组件重新渲染 → 显示新语言文本

### 初始化流程

1. main.ts 加载默认语言（zh-CN）
2. App.vue mounted → 从 storage 获取 config.language
3. 如果与默认不同 → 切换 i18n locale

## 错误处理与边界情况

- 语言配置不存在 → 默认 zh-CN
- 无效语言值 → 回退 zh-CN
- 语言文件加载失败 → 使用英文回退
- Element Plus locale 不支持该语言 → UI 组件保持英文

## 实现注意事项

1. **Element Plus locale 需要动态导入**，不能静态配置
2. **语言切换需要触发 Element Plus 重新渲染**
3. **确保所有硬编码文本都被替换**，避免遗漏
4. **storage 语言设置已存在**，无需修改类型定义
